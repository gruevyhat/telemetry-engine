# M1 demo — Quiet Ship, the solo trade campaign

## What this proves

This local walkthrough proves Spec §21.3's M1 acceptance line: the solo trade campaign completes 4 turns headless *and* by hand. Headless: `pnpm sim:smoke`/`pnpm sim:full` run the same trade-deck content through `packages/sim`'s campaign loop. By hand: this walkthrough drives the identical trade-deck content, market economy, and degradation ladder through the real browser UI — a real market feed backed by `marketAt`/`renderFeed` projections, real incident frames firing through the phase engine's `"generate"` step, and funds moving from committed ledger facts, not fixture text. It is a playable trade game, not a scripted demo.

## Clean-checkout setup

Requirements: Node 20 or newer, pnpm 9, and a locally installed Chrome or Chromium browser.

```sh
pnpm install --frozen-lockfile
pnpm test
pnpm test:integration
pnpm lint
pnpm typecheck
pnpm lint:content
pnpm sim:smoke
pnpm build:stub
pnpm build:pages
pnpm test:e2e
```

All commands must pass. `pnpm test:e2e` boots the production bundle under the GitHub Pages base path, plays the full 4-turn campaign against it, and closes it automatically.

## Launch the local demo

```sh
pnpm build:pages
pnpm demo:m0
```

(`demo:m0`'s underlying command is just `vite preview` on the built bundle — the milestone name in the script is historical, not a limit on what it serves.) Open `http://127.0.0.1:4173/telemetry-engine/`. Keep the launch command running until the walkthrough is complete.

## Walkthrough

1. Confirm the shared-screen regions are visible behind a **Hand the device to Zhan** interstitial: status bar, four-beat phase track, main panel, and ship's log.
2. Select **I am Zhan**. Confirm Zhan's private view renders with no error.
3. Confirm **DOCKSIDE** is active and the main panel shows a real market feed: two lines, `machine-parts at Regina: Cr410. Current price...` and `refined-ore at Regina: Cr188. Current price...` — computed from committed `market.tick` facts via `marketAt`/`renderFeed`, not fixture text.
4. Select **Advance turn**. This resolves DOCKSIDE's `"generate"` step (draws a real incident from the trade deck) and its own literal `purchase.settled` fact; confirm `purchase.settled` appears in the public ship's log and **COMMS** becomes active.
5. Select **Advance turn** twice more. Confirm **TRANSIT** becomes active with a jump-plotted announcement and `jump.plotted` in the log, then **ARRIVAL** becomes active.
6. Select **Advance turn** again. This resolves ARRIVAL's `sale.settled` fact and auto-skips through the next turn's automatic market-seed step, landing back on **DOCKSIDE** for turn 2. Confirm `sale.settled` appears in the log and the status bar's funds are no longer `Cr0`.
7. Repeat steps 4–6 for turns 2, 3, and 4 (12 more clicks of **Advance turn**, 16 total from the start). Confirm the market feed's numbers change turn to turn (each turn seeds its own `market.tick` facts) and that at least one turn's DOCKSIDE step fires without falling back to the oracle (the degradation ladder's rung 2) — the trade deck has 10 frames and this script only fires 4 of them, so the ladder should never need to substitute.
8. Confirm **Advance turn** disables after the 16th click (the campaign's one scripted run) rather than silently starting a second, unrequested lap.
9. Confirm the browser console shows no errors at any point in the walkthrough.

## Pass criteria

- Every clean-checkout command exits successfully.
- The dockside market feed shows real, turn-varying prices — never the M0 demo's static "Dockside systems are open" text.
- All 4 turns (16 advances) complete without a blank screen, console-visible failure, or missing announcement.
- Funds move visibly across the run (public `purchase.settled`/`sale.settled` facts, not a stuck `Cr0`).
- The advance control disables at the end of the run rather than re-committing a second lap's facts.
- The degradation ladder (Spec §17) is reachable only via chaos content — confirmed by the existing chaos-content test suite (`packages/engine/src/degrade/ladder.test.ts`, `packages/content-lint/src/*`), not by anything in this by-hand walkthrough, which should never need the ladder against real content.
