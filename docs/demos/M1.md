# M1 demo — Quiet Ship, the solo trade campaign

## What this proves

This local walkthrough proves Spec §21.3's M1 acceptance line: the solo trade campaign completes 4 turns headless *and* by hand. Headless: `pnpm sim:smoke`/`pnpm sim:full` run the same trade-deck content through `packages/sim`'s campaign loop. By hand: this walkthrough drives the identical trade-deck content, market economy, skill checks, NPC crew, evidence, and degradation ladder through the real browser UI — a real market feed backed by `marketAt`/`renderFeed` projections, real incident frames firing through the phase engine's `"generate"` step, a real skill check with a player-entered roll, an NPC interrogation ladder, an evidence reveal, and funds moving from committed ledger facts, not fixture text. It is a playable trade game, not a scripted demo.

## Clean-checkout setup

Requirements: Node 20 or newer, pnpm 9, and a locally installed Chrome or Chromium browser.

```sh
pnpm install --frozen-lockfile
pnpm test
pnpm test:integration
pnpm lint
pnpm typecheck
pnpm lint:content
pnpm sim:smoke
pnpm build:stub
pnpm build:pages
pnpm test:e2e
```

All commands must pass. `pnpm test:e2e` boots the production bundle under the GitHub Pages base path, plays the full 4-turn campaign (including its check steps) against it, and closes it automatically.

## Launch the local demo

```sh
pnpm build:pages
pnpm demo:m0
```

(`demo:m0`'s underlying command is just `vite preview` on the built bundle — the milestone name in the script is historical, not a limit on what it serves.) Open `http://127.0.0.1:4173/telemetry-engine/`. Keep the launch command running until the walkthrough is complete.

## Walkthrough

1. Confirm the shared-screen regions are visible behind a **Hand the device to Zhan** interstitial: status bar, four-beat phase track, main panel, and ship's log.
2. Select **I am Zhan**. Confirm Zhan's private view renders with no error.
3. Confirm **DOCKSIDE** is active and the main panel shows a real market feed: two lines, `machine-parts at Regina: Cr410. Current price...` and `refined-ore at Regina: Cr188. Current price...` — computed from committed `market.tick` facts via `marketAt`/`renderFeed`, not fixture text.
4. Select **Advance turn**. This resolves DOCKSIDE's `"generate"` step (draws a real incident from the trade deck, naming an NPC — turn 1 is `npc:kessler`) and its own literal `purchase.settled` fact; confirm `purchase.settled` appears in the public ship's log and **COMMS** becomes active.
5. At COMMS, confirm two optional controls appear: **Persuade npc:kessler** / **Intimidate npc:kessler**, and **Investigate**. Select **Persuade npc:kessler**, enter a roll of `9` (the interrogation difficulty is 6, so this succeeds with Effect 3), and submit. Confirm an answer renders in the main panel and includes Kessler's tell line. Then select **Investigate**, enter a roll of `10` (difficulty 6, Effect 4), and submit. Confirm the main panel reports revealed fields and the public ship's log gains two `reveal` entries (non-identity fields first, the identity field `actor` only in a later entry) plus the day-cost `clock.tick` (Obligation moves to `-1d`).
6. Select **Advance turn**. This resolves COMMS (a no-op stub) and lands on the TRANSIT check step. Confirm a **Roll total** control appears (not the Advance button) — enter `9` (difficulty is 7) and select **Submit roll**. Confirm the main panel reports the check's success line ("flown clean").
7. Select **Advance turn**. This resolves the check's success branch, posts `jump.plotted`, and **ARRIVAL** becomes active.
8. Select **Advance turn** again. This resolves ARRIVAL's `sale.settled` fact and auto-skips through the next turn's automatic market-seed step, landing back on **DOCKSIDE** for turn 2. Confirm `sale.settled` appears in the log and the status bar's funds are no longer `Cr0`.
9. Repeat turns 2–4 (DOCKSIDE → COMMS, optionally interrogate/investigate → the TRANSIT check roll → ARRIVAL), 20 advances total from the start (5 per turn: DOCKSIDE, COMMS, the check, its branch, ARRIVAL — interrogation/evidence rolls are optional side actions and don't count toward this). Confirm the market feed's numbers change turn to turn and that every DOCKSIDE `"generate"` step fires a real trade-deck incident without falling back to the oracle (the degradation ladder's rung 2) — the trade deck has 10 frames and this script only fires 4 of them, so the ladder should never need to substitute.
10. Confirm **Advance turn** disables after the 20th click (the campaign's one scripted run) rather than silently starting a second, unrequested lap.
11. Confirm the browser console shows no errors at any point in the walkthrough.

## Pass criteria

- Every clean-checkout command exits successfully.
- The dockside market feed shows real, turn-varying prices — never the M0 demo's static "Dockside systems are open" text.
- All 4 turns (20 advances, plus optional interrogation/evidence side actions) complete without a blank screen, console-visible failure, or missing announcement.
- Funds move visibly across the run (public `purchase.settled`/`sale.settled` facts, not a stuck `Cr0`).
- The TRANSIT check accepts a player-entered roll (never rolls for the player, Spec §6) and takes the correct onSuccess/onFail branch.
- The interrogation control commits `npc.statement` (table-visibility) and `npc.truthTierAssigned` (referee-visibility) per fact-kinds-v0.md §3, neither of which appears in the public ship's log (only `public`-visibility facts do); the rendered answer text itself is presentation only.
- The evidence control never shows a roll-entry control when access is denied (no roll, no day cost on denial, Spec §10.1); on access-granted, it commits `reveal` fact(s) and the atomic day-cost `clock.tick`, with the identity field (`actor`) never revealed before the target's other fields are exhausted.
- The advance control disables at the end of the run rather than re-committing a second lap's facts.
- The degradation ladder (Spec §17) never fires during this walkthrough. This is a structural guarantee, not a test claim: `phases/interpreter.ts`'s `"generate"` step only calls `runDegradeLadder` on the branch where the script's named `gen.frameId` is missing from the supplied deck, and every `"generate"` step in `content/frames/trade-campaign/turn.json` names a real id present in `content/decks/trade/frames.json`. The ladder's own reachability *when* content is broken is separately covered by `packages/engine/src/degrade/ladder.test.ts`'s chaos-content suite.

## Known gaps (not covered by this walkthrough)

- Comms-window ordering/agendas, confrontations, and envelopes are M2 — COMMS stays a pass-through stub here.
- The interrogation and evidence controls target only the three NPCs (`npc:kessler`, `npc:reyes`, `npc:okonkwo`) and one hardcoded evidence query (the aft-bay lock log) this script's four incidents can name; there is no general "any NPC, any evidence query" composer yet.
- The phase engine's `"generate"`/oracle rendering still uses `interpreter.ts`'s internal stub, not M1-09's real `createTemplateRenderer` — flagged as a follow-on in M1-13, unchanged since.
