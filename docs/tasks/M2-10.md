# M2-10 — Referee-payload encryption at rest

## Prerequisite
M2-00 must record owner approval for the save-schema version and migration behavior before this schema-changing task begins.

## Tests first
- persistence: exported/IndexedDB bytes contain no known referee plaintext or private preimage
- replay: correct reconstructed key restores byte-identical facts; wrong key and tampering fail loudly
- migration: existing M1 saves either load through an explicit migration or fail with an actionable version error

## Done when
- referee payloads use authenticated platform encryption with a key derived per §16 and held in the sealed runtime context
- export/import preserves the material needed for authorized reconstruction and black-box release

## Do not
- no new engine dependency and no silent plaintext compatibility fallback
- do not describe encryption as malicious-host-proof
