# M2-12 — Comms phone UI and host-authoritative timer

## Tests first
- component: every phone shows the same shell and lock duration while only an agenda holder sees an action menu
- integration: selected action queues once; routine/negative-deal clients submit indistinguishable acknowledgements
- timing: early acknowledgement is ignored and re-locked; only host close releases all clients

## Done when
- paired phones run the thirty-second silent ritual, submit M2-05 actions, and show private fizzle feedback next window
- the shared display reveals neither action count nor early completion

## Do not
- no local client authority over timer close and no UI difference that reveals agenda status at a glance
- no spoken-play recording or evaluation
