# M2-04 — Agenda actions and incident claimant seam

## Tests first
- property: a claimed frame and its innocent twin produce byte-identical surface descriptors
- property: every surfaced claimed incident retains at least one visible innocent explanation
- unit: access/timeline failure returns a typed fizzle proposal and no action effects

## Done when
- pure agenda evaluation emits `agenda.actionTaken` and action-specific proposals; it never writes the ledger
- matching queued actions claim frames via the shipped `claimant.agendaActionId` seam; unmatched frames instantiate twins

## Do not
- no prose in proposals and no validator bypass for agenda work
- do not resolve actions live during COMMS
