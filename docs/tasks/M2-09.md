# M2-09 — Transport contracts and scoped encrypted envelopes

## Tests first
- fuzz: no referee or foreign-private fact appears in any serialized client payload
- property: a payload encrypted for one player cannot be decoded with another player's key
- unit: duplicate/out-of-order message ids are idempotent or rejected deterministically

## Done when
- a transport-independent host/client protocol carries versioned public state, one player's private slice, actions, acknowledgements, pause, and reconnect snapshots
- the host is the only full-ledger holder; adapters cannot accept a full ledger as client payload

## Do not
- no WebRTC or UI in this card and no unencrypted private fallback message
- do not claim protection from a deliberately instrumented host
