# M1-11a — Content format v1 + trade deck (regenerates incident-cards-spec)

## Carried over from M1-10 (degradation ladder)
M1-10 built `packages/engine/src/degrade/ladder.ts`'s `runDegradeLadder()` (the rung 1-4
mechanism, Spec §17) and unit-tested it against injected `attemptGeneric`/`attemptOracle`
functions, but **did not wire it in anywhere** — there is no real curated-deck composer
(weighted eligible-pool selection over `generate/frame.ts`'s `eligibleFrames`/`CooldownState`,
validated per-candidate) for rung 1's "composer exhaustion" to trigger from yet; that composer is
this task's own deliverable. Right now `phases/interpreter.ts`'s `resolveStep`'s `"generate"`
case still `throw`s when a named frame is missing from the deck — INV-14 ("from any reachable
state, the engine produces a next playable step") is **not enforced on any reachable path**
today. This task should: (a) build the real composer, (b) have it hand off to
`runDegradeLadder()` instead of throwing when the pool is exhausted, and (c) include the
deck-driven chaos suite M1-10's own card asked for ("chaos suite: decks engineered to fail each
rung") using real (deliberately-broken) decks through the real composer, not injected lambdas.

## Tests first
- content-lint: schema + referential + balance passes on the deck
- sim:smoke on the deck within thresholds
- chaos suite: a deliberately-broken deck (all frames invalid/on cooldown) drives the real
  composer to rung 1 via `runDegradeLadder()`; a normal, healthy deck never triggers it

## Done when
- incident-content format v1 doc (supersedes incident-cards-spec.md: four pillars, claimant field, access preconditions on every evidence entry)
- trade deck: ≥10 frames including the three legacy trade cards updated
- slot tables for trade
- the curated-deck composer (weighted eligible-pool selection + per-candidate validation) exists
  and hands off to M1-10's `runDegradeLadder()` on exhaustion, closing the INV-14 gap above

## Do not
- Espionage/exploration/warfare decks are M4/M5 content — format must accommodate them, files need not exist
- No verbatim Traveller table prose (NOTICE.traveller rules)
