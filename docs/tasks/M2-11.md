# M2-11 — WebRTC and QR phone pairing

## Prerequisite
M2-00 must approve trystero/QR UI dependencies; none may enter `packages/engine`.

## Tests first
- integration: host creates a room, two clients pair to distinct player identities, and reconnect snapshots converge
- security: swapping or misrouting signaling/application payloads never reveals readable foreign-private data
- component: QR and manual pairing code represent the same room/player/key material

## Done when
- M2-09 has a trystero adapter and the phone build pairs from QR without receiving the full ledger
- pairing errors are recoverable and never expose keys or private payloads on the shared screen

## Do not
- no server backend, account system, or promise of internet-offline signaling
