# M2-13 — Disconnect pause, resume, and hotseat fallback

## Tests first
- e2e: forced disconnect mid-window pauses the host timer and prevents partial batch resolution
- integration: reconnect resumes the same remaining window and yields the same facts as uninterrupted play
- e2e: declined/failed reconnect offers hotseat and completes without exposing another player's private slice

## Done when
- transport loss produces a visible paused state, autosaves, and offers reconnect or the existing hand-to interstitial
- no queued action is lost, duplicated, or resolved before the window closes

## Do not
- no countdown while a required client is disconnected and no blank/error screen
