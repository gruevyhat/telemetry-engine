# M1-15 ‚Äî NPC interrogation ladder UI

## Context

`docs/design/screens-v1.md` ¬ß2.2 designed this screen for M1 but no card ever built it; M1-13
scoped it out explicitly to stay under the diff-size split point. Unlike M1-13's dockside
market/incident wiring, this is a **player-initiated, mid-beat action** (Persuade/Intimidate on an
NPC, reachable during COMMS or DOCKSIDE) ‚Äî it cannot be pre-scripted as a script's literal `facts`
the way M1-13's buy/sell were, because the player chooses when and whom to interrogate, and the
resulting `check.reported` fact and the NPC's ladder-tiered answer depend on that live choice.

INV-6 ("nothing writes to the ledger except the phase-engine interpreter") means only
`createPhaseInterpreter`'s `advance()` may append facts. Today's interpreter has no API for "commit
a player-chosen action inside the current beat" ‚Äî `queueCommsAction` exists but is an inert M2
stub (`commsWindow` step kind is a no-op batch-resolution stub). Two materially different designs
both fit the Spec's silence on this:
1. Model interrogation as its own `branch`+`check` sub-sequence of script steps (stays inside the
   existing step-kind vocabulary, but a script can't express "any NPC, chosen at play time" without
   either enumerating every NPC as separate steps or a templated/generated sub-script).
2. Add a new interpreter action API (e.g. an `interrogate(actor, npcId, approach, checkTotal)`
   method alongside `advance`) that appends its own transaction directly, still gated through the
   interpreter (INV-6 preserved) but a new surface distinct from `advance`/`StepInput`.

## Stop ‚Äî this needs owner sign-off before implementation

Per CLAUDE.md: "The Spec is silent and two materially different implementations both fit the
nearest *Why*" is a stop-and-ask condition. Do not pick one and build it. Bring both options (and
any better one found while reading `packages/engine/src/npc/{policy,interrogation}.ts`, which
already implement the truth-ladder logic this UI would call) to the owner before writing code.
