M2 joins secret agenda mechanics, two UIs, transport, timers, confrontation, and cryptographic audit. Before code, record their boundaries in `docs/design/screens-v2.md` and close the decisions below. **Forced-vote prerequisite approved 2026-07-18:** an envelope is burned only by a carried majority confrontation vote, never voluntarily. The Spec, rulebook, fact catalog, and Appendix A now record that authority for M2-07.

## Owner decisions required
- **Agenda content — approved 2026-07-18.** Use the declarative shape in screens-v2 §2.4: id + template key, access precondition, optional conjunctive target selector restricted to a finite non-empty exact-kind list, fact-proposal templates with a closed value-reference vocabulary, action-specific implication metadata, and numeric payout/exposure metadata. Incident frames continue to claim by `agendaActionId`; no executable content callbacks.
- **Durable queue and fact catalog — approved 2026-07-18.** `agenda.actionTaken` means append-only queued selection/replacement, with window, target, and client-command ids, rehydrated by pinned content id at close. Add catalog-first kinds for private objective assignment, deferred-reveal mint/cash, captain assignment, and crew removal; revise `vote.recorded` with immutable eligibility/threshold/ballot state; use existing `confrontation.resolved` for let-it-lie.
- **UI dependencies — approved 2026-07-18.** Use Spec-prescribed `trystero` in a WebRTC adapter package and browser `qrcode` plus TypeScript definitions in `ui-shared`. No dependency enters `packages/engine`.
- **Encrypted persistence — approved 2026-07-18.** Save schema v2 has a one-way v1 migration, a wrapped—not plaintext—campaign seed, a random recovery key stored locally as non-extractable Web Crypto material, and a one-time portable recovery code/download. Persist binding epochs, replay counters, and timer checkpoints, never live peer ids, QR tokens, or phone keys. Phone/host reload re-pairs; ordinary disconnect retains the in-memory key.
- **Package boundary — approved 2026-07-18.** Use `packages/transport` (protocol/crypto, no engine or React), `packages/transport-webrtc` (trystero adapter), a host bridge in `ui-shared`, and a thin consumer in `ui-phone`. An engine-owned `buildPlayerDelivery` API is the only authorized referee-to-private-presentation projector; the host bridge may serialize only its DTOs.
- **Confrontation ballot visibility — approved 2026-07-18.** Final ballots become public as cast, matching `sim-bot-policies.md`'s `majority-so-far` rule. The accused votes; a strict majority of the fixed active-PC eligibility set carries; only COMMS actions remain sealed until close.
- **Commit/reveal binding — approved 2026-07-18.** `te-commit-v1` first commits the campaign seed, then binds every secret draw's seed commitment, stream, index, canonical result, and salt. The black box independently verifies the seed hash, re-derived RNG result, and draw hash; changed results must fail.
