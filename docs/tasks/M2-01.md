# M2-01 — Secret-draw commit/reveal primitive

## Tests first
- property: the campaign-seed commitment rejects any changed scheme, seed, or campaign salt
- property: the draw commitment rejects any changed scheme, seed-commitment hash, stream, index, canonical result, or draw salt
- property: object key insertion order does not change canonical result bytes; array order and every finite JSON value do
- property: identical seed/context produces byte-identical commitment and reveal records after restart
- unit: public commitment contains only scheme/link metadata and the hash; seed, result, salts, and other preimage fields remain outside every live scoped view

## Done when
- a framework-free engine API produces the setup-time `campaign.seedCommitted` proposal, secret draw result, linked `secretRoll.committed` proposal, and black-box preimage records
- `te-commit-v1` uses SHA-256 over domain-separated, length-prefixed UTF-8 fields and the Spec §6 canonical-JSON contract
- hashing uses the platform cryptographic API with no new engine dependency

## Do not
- no agenda dealing or existing draw-site migration here; M2-02/M2-03 consume the primitive
- no plaintext preimage in a live ledger fact or renderer input
